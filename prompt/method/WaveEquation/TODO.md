## 1. 模块命名

### 当前名称
- `WaveEncoderBlockV2` （临时名称，不够贴切）

### 推荐名称方案（按优先级）

#### 方案A：**ASWB** (Adaptive Scene Wave Block) - 自适应场景波动块 ⭐推荐
- **优势**：
  - 强调"场景自适应"，符合密集/稀疏场景的核心动机
  - 简洁易记，4字母缩写符合学术惯例
  - "Scene"明确指向实际应用场景（田间密度变化）
- **中文全称**：自适应场景波动编码器
- **LaTeX宏定义**：
  ```latex
  \newcommand{\ASWB}{ASWB }
  \newcommand{\ASWBfull}{Adaptive Scene Wave Block }
  ```

#### 方案B：**DPWE** (Dynamic-Parameter Wave Encoder) - 动态参数波动编码器
- **优势**：直接体现"动态参数"这一技术特点
- **劣势**：相比A方案，缺少"场景"的实际应用导向

#### 方案C：**SAWM** (Scene-Aware Wave Module) - 场景感知波动模块
- **优势**：语义清晰
- **劣势**："Aware"不如"Adaptive"强调主动调节

### 🎯 最终建议
**采用方案A：ASWB (Adaptive Scene Wave Block)**

**理由**：
1. 符合论文的核心故事线："不同场景（密集/稀疏）需要不同的特征处理策略"
2. 与数据集挑战直接关联：GWHD的密度差异（12-118个/图）
3. 易于在Introduction中铺垫："In real-world panicle detection scenarios, field scenes vary dramatically..."

```

1. 目标边缘高频信息密集（相邻麦穗间距<5像素）
2. 边界模糊导致的定位困难（IoU下降10-15%）

物理需求：
- 小α：保留高频边缘信息，防止边界模糊
- 大c：快速聚合局部上下文，区分相邻目标
```

```
物理需求：
- 大α：平滑背景噪声，抑制假阳性
- 小c：减缓传播，避免背景特征扩散到目标区域
```


**公式**：
```latex
\begin{equation}
\begin{aligned}
\text{params} &= \text{Sigmoid}(\text{MLP}(\text{AdaptiveAvgPool2d}(x))) \\
\alpha &= \alpha_{\min} + \text{params}_{0} \cdot (\alpha_{\max} - \alpha_{\min}) \\
c &= c_{\min} + \text{params}_{1} \cdot (c_{\max} - c_{\min})
\end{aligned}
\end{equation}
```
